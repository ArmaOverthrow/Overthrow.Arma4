Detailed Description

Manages player data persistence and access across the game session. Handles mapping between player IDs and persistent IDs, storing player-specific data like money, home location, skills, etc. Also responsible for replicating player data to clients joining in progress (JIP).

override bool OVT_PlayerManagerComponent::RplLoad ( ScriptBitReader reader)

Loads the state of players received via replication (e.g., for JIP clients). Reads player count, then iterates reading persistent ID and runtime ID. If the player data doesn't exist locally, it's created. Populates the OVT_PlayerData fields from the stream and updates ID mappings. Finally notifies the skill system.

Returns
