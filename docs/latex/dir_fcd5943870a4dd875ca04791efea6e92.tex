The persistence system in Overthrow uses EPF\textquotesingle{}s serialization methods to convert game objects and states into data that can be saved to disk. When players reconnect to a server or load a saved game, this data is deserialized and used to restore the game world to its previous state.

Each save data class defines what specific data should be serialized and how it should be reconstructed during loading. This modular approach allows different game systems to manage their own persistence needs while still maintaining a unified save/load process through the central EPF Persistence Manager.\hypertarget{README.md_autotoc_md77}{}\doxysubsection{\texorpdfstring{Enfusion Persistence Framework}{Enfusion Persistence Framework}}\label{README.md_autotoc_md77}
Overthrow implements the \href{https://github.com/Arkensor/EnfusionPersistenceFramework/}{\texttt{ Enfusion Persistence Framework}} which provides\+:


\begin{DoxyItemize}
\item {\bfseries{Game Mode Integration}}\+: Framework hooks into the Overthrow game mode lifecycle
\item {\bfseries{Persistence Manager}}\+: Central coordination of saving and loading operations
\item {\bfseries{Component Setup}}\+: Structure for making entities and components persistent
\item {\bfseries{World Entity Save-\/\+Data}}\+: System for tracking and restoring entities in the world
\item {\bfseries{Custom Component Save-\/\+Data}}\+: Support for Overthrow\textquotesingle{}s unique component data
\item {\bfseries{Versioning}}\+: Handling data structure changes between game versions
\item {\bfseries{Utilities}}\+: Tools for managing persistence data and selective loading
\end{DoxyItemize}

For detailed information on how EPF works and how to extend it, refer to the \href{https://github.com/Arkensor/EnfusionPersistenceFramework/blob/armareforger/docs/index.md}{\texttt{ EPF Documentation}}.
